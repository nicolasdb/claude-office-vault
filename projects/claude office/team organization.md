### The Speed Problem

AI lets us make an entire application in 5 days instead of months. That’s good if you’re the only one working on it, but when there are others on the same project the typical issues of coordination become much more obvious. The code can change faster than ever, ideas can move just as fast. This is part of what is preventing teams from rolling out AI as fast as they would like.

### The Coordination Gap

Some sort of common vision is needed, daily calls are almost a requirement but you can’t possibly talk about all that you’re doing. Who is going to maintain the documentation when you can make a whole feature in the same time? Yet over time even a great team will suffer some consequences.

### What Teams Actually Need

Teams need some way to help them that works with the tools they use themselves and not outside of them. They need something they don’t need to overly worry about managing manually and updating.
- An automated note taking AI that resumes sessions.
- Github issues that are dynamic.
- A conversation app with dedicated channels.
- A very communicative company culture
- But also something else.

### Introducing Claude Office

That something else, we made it in the form of a Claude Code plugin: Claude Office. It has a hook that automatically logs coding sessions once completed, provided you have it active. It also has a command to retroactively add all your sessions.

But where to add it becomes the next question, so we made a flexible documentation system to go along with this, using md files, running with a pre-configured Obsidian; all for free and ready for a team to use!
